📄
One file to source
A single source shui.zsh bootstraps everything — tokens, themes, icons, and all fifteen components.
⚡
Zero dependencies
Pure Zsh. No npm, no Homebrew, no external packages. Works wherever Zsh 5.0+ is installed.
🎨
Semantic API
Write shui success and shui error — not escape codes. Themes handle colour; you handle meaning.

Themes

Three themes, one API

Switch themes with a single environment variable. Every component adapts automatically — no code changes required.

default 256-colour
SHUI_THEME=default

❌ Build failed
⚠️ API key expires
ℹ️  Dry-run mode

SHUI_THEME=minimal

plain no colour
SHUI_THEME=plain
[ok] Deployment complete
[!!] Build failed
[**] API key expires
[..] Dry-run mode
[ok]  [warn]  [fail]
# Combine themes and icon sets freely
SHUI_THEME=minimal SHUI_ICONS=emoji source shui.zsh
SHUI_THEME=plain   SHUI_ICONS=none source shui.zsh  # fully plain
SHUI_THEME=default SHUI_ICONS=nerd source shui.zsh  # nerd font glyphs
Install

Up and running in seconds

Works with every major Zsh plugin manager — or just clone it manually. No build step, no compilation.

Manual
git clone https://github.com/kud/shui ~/.shui
Antidote / .zsh_plugins.txt
kud/shui
Zinit
zinit light kud/shui
Submodule (recommended for scripts)
git submodule add https://github.com/kud/shui lib/shui
Oh My Zsh
Clone into custom plugins, then add shui to the plugins array in ~/.zshrc View instructions →
Quick start
# Source at the top of any Zsh script
source "${0:A:h}/lib/shui/shui.zsh"
 
# Then use components naturally
shui success "Deployment complete"
shui error "Build failed"
shui warning "Check your config"
shui info "Dry-run mode enabled"
 
# Opt into a theme or icon set
SHUI_THEME=minimal source shui.zsh
SHUI_ICONS=emoji  source shui.zsh
Requires Zsh 5.0+. Use SHUI_ICONS=emoji or none if you don't have a Nerd Font installed.
