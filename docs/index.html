Why shui

Built for real shell scripts

Stop hard-coding ANSI escape sequences. shui gives your scripts a consistent visual language with zero overhead.

⚡
One file, instant source

Drop source shui.zsh at the top of any script. Everything loads in a single pass — no package manager, no install step.

🎨
Token-based themes

Colours live in theme files, not components. Switch between default, minimal, and plain with one env var — or author your own.

🔤
Three icon sets

Ship with Nerd Font glyphs, Unicode emoji, or plain ASCII. Set SHUI_ICONS=emoji and every component adapts automatically.

🧩
Semantic API

Every component speaks intent: shui success, shui warning, shui badge error EOL. No magic numbers, no raw escape codes.

♿
NO_COLOR aware

Respects the NO_COLOR environment variable spec out of the box. CI pipelines and screen readers get clean plain-text output automatically.

🎛️
Interactive prompts

confirm, select, and input give your scripts a polished interactive layer — with sensible defaults and keyboard navigation.

Get going fast

Quick Start

Clone the repo, source the main file, and start using components immediately. 

# 1. Clone
git clone https://github.com/kud/shui.git ~/shui

# 2. Source in your script
source ~/shui/shui.zsh

# 3. Use components
shui section "Deploying to production"
shui info    "Pulling latest changes..."
shui success "Build complete"
shui warning "Restart required"

# Switch theme and icon set
SHUI_THEME=minimal SHUI_ICONS=emoji source ~/shui/shui.zsh
shui success "Works in any terminal"
Commands

Component Reference

Every component is invoked via shui <command>. All components write to stderr, keeping stdout free for piping.

shui success "msg" Green success message with a checkmark icon
shui error "msg" Red error message with an × icon
shui warning "msg" Yellow warning message with a ⚠ icon
shui info "msg" Cyan info message with an ℹ icon
shui bold | dim | italic | underline "msg" Text styling — combine with any message
shui text --color=<semantic> "msg" Semantic colour: primary success warning error info muted accent
shui section "heading" Top-level section header in primary colour
shui subtitle "text" Bold sub-section heading in warning colour
shui subsection "text" Bulleted item for nested content
shui divider [--color=<c>] [--char=<ch>] Full-width horizontal rule; customisable char and colour
shui fence "label" Labelled divider: ── label ───────────────
shui spacer [n] Print n blank lines (default 1)
shui badge <color> "label" Solid-background inline badge — use inside echo "$(shui badge success OK)"
shui pill <color> "label" Rounded-edge inline pill tag — same colours as badge
shui box [--title="T"] "content" Bordered content block; supports \n in content and inline badges
shui table "Col|Col" "row|row" … Auto-sized pipe-separated table; first arg is the header row
shui progress <val> <max> [--width=N] [--label="…"] Inline progress bar; width defaults to terminal width
shui spinner <cmd> [args…] Animated spinner while a command runs; exits with the command's exit code
shui loader [--style=dots|pulse|spinner] Indeterminate looping indicator for long-running background work
shui confirm "question?" Interactive yes/no prompt; returns exit code 0 for yes
shui select "prompt" opt1 opt2 … Arrow-key selection menu; prints chosen value to stdout
shui input [--default="val"] "prompt" Text input with optional default; prints entered value to stdout
shui screen "heading" <cmd> [args…] Full-screen section runner — prints header, runs command, shows elapsed time
shui animation typewriter|fade-in "text" One-shot text effects for intro sequences or splash screens
Configuration

Environment Variables

Set these before sourcing shui.zsh to customise behaviour across all components.

Variable Values Default Description
SHUI_THEME default minimal plain default Active colour theme. minimal uses 16-colour ANSI; plain strips all colour and uses ASCII icons.
SHUI_ICONS nerd emoji none nerd Icon set. nerd requires a Nerd Font; emoji uses Unicode emoji; none disables icons entirely.
NO_COLOR any non-empty value unset Disables all colour output when set. Follows the no-color.org spec.
SHUI_DIR directory path auto Root directory of the shui installation. Detected automatically from the sourced file path; override only if symlinked.
